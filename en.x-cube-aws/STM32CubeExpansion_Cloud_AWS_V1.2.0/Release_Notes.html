Overview

  

•  The X-CUBE-AWS (STM32 AWS IoT software expansion for STM32Cube) provides ready-to-run firmware examples to support quick evaluation and development of AWS IoT Cloud applications on B-L475E-IOT01, 32F413HDISCOVERY and 32F769IDISCOVERY boards.

•  Three IDE are supported: IAR, Keil and System Workbench.

•  It contains BSP compatible sensor drivers for B-L475E-IOT01.

•  Below the most useful document:

º  UM2178 - STM32 AWS IoT software expansion for STM32Cube.

 

Update History

  

V1.2.0 / 2-February-2018

 

Main Changes

•  Updated dependencies (see Content list below).

º  Updated to last versions of HAL, BSP, "AWS IoT device SDK for embedded C", LwIP and mbedTLS.

•  Bug fixes.

º  ES_WIFI driver (B-L475E-IOT01, 32F413HDISCOVERY): driver improvements.

–  Fix potential memory overflow happening on disconnection.

–  Improve error management on disconnection.

–  Improve SPI signals timing to speed up device communication.

–  Simplify Send and Received SPI functions.

–  Move to interrupt driver management.

–  Support Bare Metal and RTOS environment.

Known Limitations

•  Network connection instability on B-L475E-IOT01 and 32F413HDISCOVERY:

º  The board may not connect to the WiFi access point at first trial.

º  The resolution of the remote host name sometimes fails - it seems to depend on the cache status of the DNS server of the local network.

º  Remote server disconnection reported as a system error by the WiFi module.

•  The Remote Firmware Update has been obsoleted and is not provided anymore. It will be replaced and based on the Secure Boot and Secure Firmware Update component.

•  System Workbench for STM32.

º  The credentials are erased when the debugger is launched again.

º  B-L475E-IOT01 board rev B and rev C requires to use System Workbench with version greater or equal to 2.1. A debug option must be changed:

–  Right click on the project, select "Debus as...", "Debug Configuration...", the "B-L475E-IOT01.elf" target, then the "Debugger" tab.

–  If the "Reset Mode" option is not visible, click on the "Show generator options..." button.

–  The "Reset Mode" option must be set to "Software system reset".

  

V1.1.0 / 1-September-2017

 

Main Changes

•  Updated dependencies (see contents list below).

•  Bug fixes.

º  ES_WIFI driver (B-L475E-IOT01, 32F413H-DISCO): Solved random network disconnections that were due to a de-synchronization between the SPI and the GPIO.

º  TCP/IP configuration (32F769I-DISCO).

–  Generate a unique MAC address derived from the CPU ID in order to avoid address collisions when several F7 boards are connected to the same LAN.

–  Activate the local port number true randomization, to avoid possible TLS session confusions on server side after a board reset.

–  Activate the TCP keep-alive. Otherwise, the remote host may disconnect earlier than the first MQTT keep-alive message.

º  Improved error management.

•  Other changes.

º  Introduction of an internal network socket abstraction layer: NET API.

º  Source tree update to isolate the files which are common to all Cloud Cube packages.

º  RFU.

–  Complies to HTTP 1.1.

–  Inter-operable with the Apache web server.

–  Supports server-side connection close, and client reconnection.

–  Supports HTTPS (the root CA certificate must be provisioned together with the Cloud and time servers').

º  RTC initialization from HTTP.

–  Network load reduction: HEAD command instead of GET.

–  Supports HTTPS (the root CA certificate must be provisioned together with the Cloud and RFU servers'. See the application readme.txt for more details).

Known Limitations

º  A workaround is required to debug the B-L475E-IOT01 board with System Workbench:

–  In case of B-L475E-IOT01 board rev B or C, it is needed to uncomment line "reset_config none separate" in OpenOCD configuration file STM32L475VG.cfg provided under SW4STM32 directory.

–  Furthermore, some versions of System Workbench may have problem to read the OpenOCD configuration file if it is placed in project directory. In that case, copy STM32L475VG.cfg outside of sources directory (for example to c:\) and use it from here (in System Workbench menu "Run/Debug configuration", create or choose STM32L475 configuration, go to Debugger Tab, Select "Script/Use local script", indicate the copied script in the "Configuration Script" field).
